# Surface Defect Analyzer

**Surface Defect Analyzer** — учебное приложение для анализа изображений и поиска подозрительных дефектов поверхности.

Приложение позволяет загрузить изображение, выделить область анализа, найти возможные дефекты, показать маски и сохранить результат.

## Описание проекта

Проект реализован на Python с использованием OpenCV, NumPy, Streamlit и scikit-learn.

В приложении есть два режима анализа:

- **Одно изображение** — поиск локальных визуальных отклонений на одном изображении.
- **По нормальным примерам** — сравнение проверяемого изображения с набором нормальных изображений того же типа поверхности.

Первый режим обеспечивает быстрый анализ и подходит для простых гладких поверхностей.  
Второй режим лучше подходит для текстурированных материалов и датасетов, где есть нормальные изображения `train/good`.

## Ключевые возможности

- Загрузка изображений JPG, JPEG и PNG.
- Автоматическое выделение объекта от контрастного фона.
- Режим анализа всего изображения как объекта.
- Поиск подозрительных дефектных областей.
- Анализ по нормальным примерам.
- Построение маски объекта.
- Построение маски дефектов.
- Отображение изображения с разметкой.
- Расчёт статистики по найденным дефектам.
- Таблица найденных областей.
- Сохранение результата в папку `output`.

```bash
git clone https://github.com/YOUR_USERNAME/surface-defect-analyzer.git
cd surface-defect-analyzer
```

В PyCharm можно создать обычный **Pure Python** проект с виртуальным окружением `.venv`.

```text
http://localhost:8501
```

## Режимы анализа

### Одно изображение

Режим использует классический OpenCV-подход.

Приложение выделяет объект от фона, затем ищет внутри объекта локальные яркостные отклонения.

Подходит для:

- гладких поверхностей;
- простых пятен;
- царапин;
- сколов;
- объектов на контрастном фоне.

### По нормальным примерам

Режим использует набор нормальных изображений.

Программа разбивает изображения на патчи, извлекает признаки и сравнивает патчи проверяемого изображения с патчами нормальных примеров.

Подходит для:

- текстурированных поверхностей;
- датасетов с разделением `train/good` и `test/defect`;
- случаев, где одиночный анализ изображения даёт много ложных срабатываний.

Рекомендуется загружать от 5 до 20 нормальных изображений.

## Основные параметры

| Параметр | Описание |
|---|---|
| `Режим анализа` | выбор между одиночным анализом и анализом по нормальным примерам |
| `Анализировать всё изображение как объект` | использовать весь кадр как область анализа |
| `Чувствительность выделения объекта` | насколько строго объект отделяется от фона |
| `Размер области фона по краям` | какая часть краёв используется для оценки фона |
| `Оставлять только крупнейший объект` | анализировать только самую большую найденную область |
| `Чувствительность поиска дефектов` | насколько активно искать подозрительные области |
| `Минимальная площадь дефекта` | отсечение слишком маленьких областей |
| `Сглаживание поверхности` | масштаб локального анализа поверхности |

## Выходные данные

После анализа приложение показывает:

- исходное изображение;
- маску объекта;
- маску дефектов;
- изображение с разметкой;
- статистику анализа;
- таблицу найденных дефектов.

При сохранении результата в папке `output` создаются файлы:

```text
<timestamp>_marked_result.png
<timestamp>_object_mask.png
<timestamp>_defect_mask.png
<timestamp>_report.txt
```

## Статистика

Приложение рассчитывает:

- площадь объекта;
- количество найденных дефектов;
- суммарную площадь дефектов;
- долю дефектной области;
- площадь крупнейшего дефекта;
- среднюю площадь дефекта.

## Ограничения

Приложение является учебным проектом и не гарантирует промышленную точность.

Основные ограничения:

- результат зависит от качества изображения;
- обычный режим лучше работает на простых поверхностях;
- для сложных текстур желательно использовать нормальные примеры;
- нормальные примеры должны быть похожи по материалу, масштабу и освещению;
- сильные тени, блики и сложная геометрия могут давать ложные срабатывания;
- приложение не классифицирует тип дефекта.
