# Surface Defect Analyzer

**Surface Defect Analyzer** — учебное приложение для анализа изображений и обнаружения подозрительных дефектов поверхности объекта на контрастном фоне.

Проект реализован на Python с использованием OpenCV и Streamlit.

## Описание

Приложение позволяет загрузить изображение объекта, расположенного на контрастном однотонном фоне. После загрузки программа автоматически выделяет объект, исключает фон из анализа и ищет локальные визуальные отклонения на поверхности объекта.

Результат анализа включает:

- исходное изображение;
- маску объекта;
- маску найденных дефектов;
- изображение с разметкой дефектов;
- статистику анализа;
- таблицу найденных дефектов;
- сохранение результата в файлы.

## Основной сценарий работы

```text
Изображение
→ выделение объекта от фона
→ построение маски объекта
→ поиск локальных отклонений внутри объекта
→ построение маски дефектов
→ разметка результата
→ расчёт статистики
→ сохранение результата
```

## Возможности

В текущей версии реализовано:

1. Загрузка изображения в формате JPG, JPEG или PNG.
2. Автоматическое выделение объекта на контрастном фоне.
3. Построение бинарной маски объекта.
4. Поиск подозрительных дефектных областей только внутри объекта.
5. Построение маски дефектов.
6. Отображение результата с разметкой.
7. Расчёт статистики:
   - площадь объекта;
   - количество найденных дефектов;
   - суммарная площадь дефектов;
   - доля дефектной области;
   - площадь крупнейшего дефекта;
   - средняя площадь дефекта.
8. Таблица найденных дефектов.
9. Сохранение результата анализа в папку `output`.

## Использование

1. Запустите приложение.
2. Загрузите изображение объекта на контрастном фоне.
3. При необходимости измените параметры в боковой панели:
   - чувствительность выделения объекта;
   - размер области фона по краям;
   - чувствительность поиска дефектов;
   - минимальную площадь дефекта;
   - сглаживание поверхности.
4. Посмотрите результат анализа:
   - исходное изображение;
   - маску объекта;
   - маску дефектов;
   - изображение с разметкой.
5. Проверьте статистику и таблицу найденных дефектов.
6. Нажмите кнопку **Сохранить результат**, чтобы сохранить файлы в папку `output`.

## Параметры анализа

### Чувствительность выделения объекта

Определяет, насколько сильно пиксель должен отличаться от фона, чтобы считаться частью объекта.

Чем выше значение, тем строже отделение объекта от фона.

### Размер области фона по краям

Определяет, какая часть краёв изображения используется для оценки цвета фона.

Приложение предполагает, что фон виден по краям изображения.

### Оставлять только крупнейший объект

Если параметр включён, приложение оставляет только крупнейшую найденную область и анализирует её как основной объект.

### Чувствительность поиска дефектов

Определяет, насколько активно приложение ищет локальные отклонения на поверхности.

Чем выше значение, тем больше подозрительных областей может быть найдено.

### Минимальная площадь дефекта

Области меньше указанного значения считаются шумом и не включаются в результат.

### Сглаживание поверхности

Используется для построения сглаженной версии поверхности.

Меньшее значение помогает находить мелкие дефекты.  
Большее значение лучше подходит для поиска крупных отклонений.

## Принцип работы алгоритма

### 1. Выделение объекта

Приложение переводит изображение в цветовое пространство Lab. Затем оно берёт пиксели по краям изображения и оценивает медианный цвет фона.

После этого для каждого пикселя считается расстояние до фонового цвета. Пиксели, которые достаточно сильно отличаются от фона, считаются частью объекта.

Дальше маска очищается:

```text
threshold
→ morphology open
→ morphology close
→ выбор крупнейшей компоненты
→ заполнение дырок
```

Результат этапа — бинарная маска объекта.

### 2. Поиск дефектов

После выделения объекта приложение анализирует только пиксели внутри маски объекта.

Для поиска дефектов используется L-канал цветового пространства Lab. Алгоритм строит сглаженную версию поверхности и сравнивает её с исходной яркостной картой.

Локальные отличия считаются подозрительными областями.

Упрощённо:

```text
L-канал изображения
→ сглаживание
→ разница между исходной и сглаженной версией
→ адаптивный порог
→ фильтрация по площади
→ маска дефектов
```

Чтобы граница объекта не считалась дефектом, край объекта исключается из анализа.

### 3. Статистика

После фильтрации найденных областей приложение рассчитывает:

```text
площадь объекта
количество дефектов
суммарную площадь дефектов
долю дефектов от площади объекта
крупнейший дефект
среднюю площадь дефекта
```

## Сохранение результата

При нажатии кнопки **Сохранить результат** приложение создаёт в папке `output` файлы:

```text
<timestamp>_marked_result.png
<timestamp>_object_mask.png
<timestamp>_defect_mask.png
<timestamp>_report.txt
```

Где:

| Файл | Содержимое |
|---|---|
| `marked_result.png` | изображение с разметкой дефектов |
| `object_mask.png` | бинарная маска объекта |
| `defect_mask.png` | бинарная маска дефектов |
| `report.txt` | текстовый отчёт со статистикой |

## Ограничения

Приложение является учебным проектом и не гарантирует промышленную точность.

Основные ограничения:

1. Объект должен находиться на контрастном, относительно однотонном фоне.
2. Фон желательно должен быть виден по краям изображения.
3. На изображении желательно должен быть один основной объект.
4. Объект желательно должен находиться ближе к центру кадра.
5. Алгоритм лучше работает на гладких и умеренно текстурированных поверхностях.
6. Сильные тени, блики и сложные текстуры могут вызывать ложные срабатывания.
7. Сильно текстурированная поверхность может восприниматься как набор дефектов.
8. Слабые дефекты без заметного яркостного отличия могут быть пропущены.
9. Приложение не классифицирует тип дефекта.
10. Приложение выделяет подозрительные визуальные области, а не подтверждённые производственные дефекты.

## Примеры подходящих изображений

Лучше всего подходят изображения, где:

```text
объект хорошо отличается от фона;
фон однотонный;
фон виден по краям;
дефекты заметны визуально;
освещение относительно равномерное.
```

Примеры объектов:

- металлическая пластина с царапинами;
- пластиковая деталь с пятнами;
- круглая деталь на контрастном фоне;
- окрашенная поверхность с потёртостями;
- гладкая поверхность с загрязнениями.

## Что не входит в MVP

В текущую версию не входят:

1. нейросетевая модель;
2. обучение модели;
3. сравнение с эталонным изображением;
4. работа с видеопотоком;
5. поддержка сложного фона;
6. классификация типов дефектов;
7. база данных;
8. авторизация;
9. пакетная обработка изображений;
10. ручная коррекция маски объекта.
